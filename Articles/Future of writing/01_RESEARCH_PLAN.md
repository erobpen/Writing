# Research Plan

## Purpose

This file tracks the research needed to turn `00_SPEC.md` into a publishable article for Medium and, potentially, Towards AI.

The goal is not to collect sources for decoration. The goal is to reduce hallucination risk, support factual claims, and identify which claims are original interpretation rather than established fact.

## Research Priorities

1. Verify publication rules for Towards AI.
2. Support the historical framing of writing tools and language models.
3. Support the claim that human-computer communication has been a long-standing goal.
4. Verify claims about VS Code, Git, GitHub, and Codex.
5. Track the author's own project repository as a transparency reference.
6. Verify or replace the YouTube reference candidate before final publication.

## Claim Groups

### Writing Tools Evolved Over Time

Claims to support:

- Typewriters are part of the long evolution of writing tools.
- Spellcheckers, grammar checkers, AutoCorrect, and autocomplete were earlier forms of computer-assisted writing.
- Earlier language models existed before modern LLMs.

Likely source types:

- Historical sources for typewriters.
- Microsoft documentation for spelling, grammar, and AutoCorrect.
- NLP textbook or paper sources for language models.

### LLMs Changed the Interface

Claims to support:

- LLMs allow tasks to be specified through natural-language interaction.
- GPT-style systems demonstrated few-shot task behavior using text prompts.
- Natural-language computer interaction can feel conversational but does not make the computer human.

Likely source types:

- LLM papers.
- Human-computer interaction papers.
- Official product documentation when referring to specific tools.

### SDA Is a Proposed Workflow

Claims to support:

- SDA is the author's proposed workflow, not an established industry standard unless a source is later found.
- The method should be framed as the author's interpretation and experiment.
- The value claim is that specification first, draft second can preserve author intent.

Likely source types:

- This repository.
- This specification.
- The later article development record.

### Setup Claims

Claims to support:

- VS Code can work with Git through an integrated source-control interface.
- Git records changes so earlier versions can be recovered.
- GitHub repositories store files and revision history.
- Codex can work in the IDE and with local project context.

Likely source types:

- VS Code documentation.
- Git book.
- GitHub documentation.
- OpenAI Codex documentation.

## Verification Rules

- Use primary or official sources where possible.
- Prefer direct links to documents, product documentation, papers, or guidelines.
- Do not cite a source that has not been opened or otherwise verified.
- If a source was provided by the author but cannot be independently verified, mark it clearly.
- If no credible source is found for a claim, rewrite the claim as interpretation, hypothesis, or personal workflow experience.
